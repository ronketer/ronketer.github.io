## Generative AI Security & Forensics

**Technologies:** Python, Stable Diffusion, LoRA, PyTorch

This project explores the intersection of generative AI and security, focusing on detecting training data leakage and optimizing image generation.

## Project Overview

As generative AI models become increasingly prevalent, understanding their security implications is crucial. This project addresses two key areas:

1. **Security Analysis:** Developing methods to detect whether specific data was used to train a generative model
2. **Model Optimization:** Fine-tuning diffusion models for high-fidelity image generation

## Key Achievements

### Membership Inference Algorithm

- Developed a novel algorithm in Python to detect training data leakage
- Established baseline security metrics for evaluating generative AI models
- Created frameworks for assessing model vulnerability to data extraction attacks

### Fine-Tuning with LoRA

- Optimized Stable Diffusion 1.5 for high-fidelity image generation
- Applied Low-Rank Adaptation (LoRA) for efficient fine-tuning
- Executed data-driven tuning cycles to improve output quality

## Technical Implementation

The project leverages PyTorch for deep learning operations and implements custom inference pipelines for membership detection. The LoRA fine-tuning approach allows for efficient model customization without requiring full model retraining.

## Security Implications

Understanding membership inference is critical for:

- Protecting privacy in AI-generated content
- Auditing AI models for compliance
- Developing more robust training procedures
